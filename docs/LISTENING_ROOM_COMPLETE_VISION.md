# THE LISTENING ROOM - Complete Vision Document

## Overview

"The Listening Room" is an AI-powered web art installation that transforms brightears.onrender.com into an immersive, mystical experience exploring the intersection of sound, color, emotion, and atmosphere. It serves as both expressionist digital art and a subtle bridge to Bright Ears' core business: providing professional DJs and atmosphere design for high-end Bangkok venues.

**Status:** In Development
**Target Launch:** 8 weeks from initiation
**Monthly Cost:** $0 (using free-tier services)
**Target Audience:** Curious explorers, art enthusiasts, venue managers, potential clients

## The Complete User Journey

### ACT I: ARRIVAL (0-15 seconds)

**Visual Experience:**
- User lands on completely dark screen
- Bright Ears logo fades in slowly (cyan glow, 2-3 seconds)
- Logo animates: gentle pulse, as if breathing
- Logo gradually shrinks and moves to bottom-right corner
- Becomes watermark at 10% opacity
- Background remains deep black (#030129)

**Audio Experience:**
- Silence initially (respecting web audio policy)
- Small speaker icon appears (bottom-left)
- User clicks to enable sound
- Single sustained tone begins (A2, 110 Hz, sine wave)
- Soft, ambient, barely noticeable

**First Interaction:**
- Mystical prompt fades in center screen:
  - "What color lives between silence and sound?"
  - OR: "If this moment had a frequency, what would it hum?"
  - OR: "Describe a sound you remember but never heard"
- Text input field appears below (minimal design)
- Brand particles (cyan, teal, brown, lavender) begin appearing
- Particles move slowly, organically, like fireflies

**Design Details:**
- Font: Playfair Display for prompt (serif, mystical)
- Text color: Soft white with subtle glow (#FFFFFF with 2px blur)
- Input field: Transparent with bottom border only
- Cursor: Blinking cyan (#00bbe4)
- No navigation, no buttons, no distractions
- Fullscreen immersion

---

### ACT II: ENIGMATIC DIALOGUE (15-90 seconds)

**User Types Response:**
- Each keystroke triggers subtle particle reactions
- Particles gravitate toward typing cursor
- Soft typing sounds (optional, very subtle)

**AI Response Process:**
1. Brief "thinking" animation (particles swirl)
2. AI response appears with typewriter effect (80ms per character)
3. Response is cryptic, poetic, never literal:
   - "If this moment were a frequency, what note would it hum?"
   - "Describe a sound you remember but never heard"
   - "What does the space between two breaths look like?"
4. New question emerges organically from user's answer

**Mood Detection (Invisible):**
- NLP analyzes user's text for keywords
- **Subtle mode** (default): Requires 3-4 mood keywords before transitioning
- **Sensitive mode** (fallback): Activates if < 3 keywords detected, uses 1-2
- Detects: energetic, romantic, happy, calming, partying
- Confidence score calculated (0-1 range)
- No visible indication mood was detected

**If Mood Detected (50%+ confidence):**
- Visual transition begins (3-4 seconds)
- Colors gradually shift to mood palette
- Particle behavior changes
  - Energetic: Faster, more chaotic
  - Romantic: Slower, flowing, paired movements
  - Happy: Bouncy, celebratory patterns
  - Calming: Gentle drift, slow waves
  - Partying: Pulsing, bass-synchronized
- Audio soundscape evolves
  - BPM adjusts (60-140 range)
  - Frequencies shift (bass/mid/treble balance)
  - Harmonies change (major/minor keys)

**No Mood Detected:**
- AI uses mood-agnostic cryptic prompts (Oblique Strategies style):
  - "Ask your body what color it needs"
  - "Honor the silence as hidden rhythm"
  - "What wouldn't you hear if you were listening?"
- Visuals remain in default "mystery" state (mixed palette)
- Audio stays ambient and neutral

### ACT III: IMMERSION (1-5 minutes)

AI adapts questions based on detected mood (if any):

**For "Energetic" Mood:**

**For "Happy" Mood:**

**For "Calming" Mood:**

**For "Partying" Mood:**
- "What color is the feeling of letting go?"
- "If bass could paint the air, what would it draw?"
- "Describe the sound of complete freedom"

**Interactive Elements:**

**Mouse Movement:**
- Particles follow cursor with attraction force
- Create "wake" trail behind mouse
- Closer particles respond more strongly
- Physics: spring forces, friction, momentum

**Mouse Clicks:**
- Ripple effect radiates from click point
- Sound trigger: harmonic tone based on current mood
- Particles scatter then regroup
- Visual bloom effect (brief color burst)

**Scrolling (optional):**
- Subtle depth effect (parallax layers)
- Particles in foreground move faster
- Creates 3D illusion

**Typing:**
- Each keystroke creates small particle pulse
- Text appears with soft glow
- Character limit: 500 (encourages thoughtful, concise responses)

**Visual-Audio Synchronization:**

- **Real-time FFT analysis:** Audio spectrum ‚Üí particle size/speed
- **Frequency-to-color mapping:** Bass (red) ‚Üí Mid (green) ‚Üí Treble (blue)
- **Amplitude-to-brightness:** Louder = brighter particles
- **BPM-to-motion:** Faster tempo = faster particle movement
- **Synesthesia principles:** Scientific mappings from research

**Learning & Adaptation:**
- AI remembers conversation context (stored client-side)
- Adjusts questions based on user's language style
- Never repeats the same cryptic prompt twice
- Gradually builds understanding of user's perspective

## Technical Constraints & Considerations

### API Rate Limits:
- **Gemini Free Tier:** 1,500 requests/day, 15 requests/minute
- **Strategy:** Client-side rate limiting (1 request per 5 seconds)
- **Fallback:** If limit exceeded, show poetic message and continue visual experience
- **Monitoring:** Daily usage dashboard

### Browser Limitations:
- **Web Audio Policy:** Requires user gesture to enable sound
- **Strategy:** Muted start + speaker icon to activate
- **Mobile Safari:** Additional audio context resume needed
- **Fallback:** Visual-only experience still compelling

### Performance Optimization:
- **Particle count:** Adjust based on device detection
- **WebGL acceleration:** Use when available
- **Canvas fallback:** For older browsers
- **Lazy loading:** Assets load progressively

### Privacy & Data:
- **No tracking cookies:** Uses localStorage only
- **No personal data:** Conversations stored client-side temporarily
- **No user accounts:** Completely anonymous
- **GDPR compliant:** No data leaves browser except API calls
